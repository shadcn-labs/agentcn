The **Weather Agent** is the classic starter: ask about a place and it reports the
current conditions. It geocodes the location and fetches live weather from the
free Open-Meteo API — no key required. Reach for it as a minimal, end-to-end
example of an agent with a single tool.

<DocsBaseSwitcher base="eve" agent="weather" />

The **Eve** recipe ships one `get_weather` tool: it geocodes the location, fetches
the current forecast, and the agent reports it conversationally.

## Install

```bash
npx shadcn@latest add https://agentcn.vercel.app/r/eve/weather
```

## Prerequisites

- An [Eve](https://vercel.com/docs/eve) project (`npx eve@latest init my-agent`).
  The recipe installs files under `agent/`, which Eve discovers automatically.
- The `eve` package (installed automatically as a dependency of the recipe).
- No API key — Open-Meteo is free and unauthenticated.
- For the **live preview below**, set `EVE_PREVIEW_URL` to a deployed Eve
  instance of this recipe. See the [Preview Backend](https://github.com/shadcn-labs/agentcn#preview-backend)
  notes in the README.

## Customization

- **Add a forecast.** Extend `get_weather` to request daily/hourly data.
- **Add units.** Pass a `units` parameter (metric/imperial) to the API.
- **Add more tools.** This is a good base to graduate into a travel or planning
  agent.
- **Swap the model.** Edit `agent/agent.ts`.
