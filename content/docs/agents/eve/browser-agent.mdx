The **Browser Agent** completes tasks on the live web by driving a real Chromium
browser. It uses a snapshot-and-selector loop — the model sees a stable list of
interactive elements instead of brittle CSS — to navigate, click, and type. Reach
for it to automate flows, scrape interactive pages, or test UIs.

<DocsBaseSwitcher base="eve" agent="browser-agent" />

The **Eve** recipe ships `browser_goto`, `browser_snapshot`, `browser_click`, and
`browser_type` over a shared Playwright page. Each action returns a fresh snapshot
so the agent always acts on the current state.

## Install

```bash
npx shadcn@latest add https://agentcn.vercel.app/r/eve/browser-agent
```

## Prerequisites

- An [Eve](https://vercel.com/docs/eve) project (`npx eve@latest init my-agent`).
  The recipe installs files under `agent/`, which Eve discovers automatically.
- The `playwright` dependency installs automatically with the shadcn CLI. Run
  `npx playwright install chromium` once, or set `BROWSER_CDP_URL` to a hosted
  Chrome/Browserbase/Browserless instance for server deployments.
- For the **live preview below**, set `EVE_PREVIEW_URL` to a deployed Eve
  instance of this recipe. See the [Preview Backend](https://github.com/shadcn-labs/agentcn#preview-backend)
  notes in the README.

## Customization

- **Use hosted browsers.** Set `BROWSER_CDP_URL` to run against Browserbase or a
  remote Chrome.
- **Richer snapshots.** Extend `lib/browser.ts` to include roles, ARIA, or
  bounding boxes.
- **Add actions.** Add tools for scrolling, file uploads, or screenshots.
- **Swap the model.** Edit `agent/agent.ts`.
