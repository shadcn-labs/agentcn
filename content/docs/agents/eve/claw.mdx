**Claw** is an autonomous assistant that operates a sandboxed workspace to finish
multi-step tasks. It reads and writes files, runs shell commands scoped to a
workspace directory, and follows reusable skills. Reach for it as a base for
coding assistants, repo automation, or any agent that needs a real filesystem.

<DocsBaseSwitcher base="eve" agent="claw" />

The **Eve** recipe ships `read_file`, `write_file`, and `run_shell` tools — all
guarded so every path stays inside the workspace — plus a `workspace` skill that
encodes a safe plan-inspect-act-verify loop.

## Install

```bash
npx shadcn@latest add https://agentcn.vercel.app/r/eve/claw
```

## Prerequisites

- An [Eve](https://vercel.com/docs/eve) project (`npx eve@latest init my-agent`).
  The recipe installs files under `agent/`, which Eve discovers automatically.
- The `eve` package (installed automatically as a dependency of the recipe).
- A `WORKSPACE_DIR` (defaults to `./workspace`). The sandbox guard refuses any
  path that escapes it.
- For the **live preview below**, set `EVE_PREVIEW_URL` to a deployed Eve
  instance of this recipe. See the [Preview Backend](https://github.com/shadcn-labs/agentcn#preview-backend)
  notes in the README.

## Customization

- **Harden the sandbox.** Run `run_shell` inside a container or jail; the path
  guard only constrains file paths, not arbitrary commands.
- **Add web browsing.** Pair with the [Browser Agent](/docs/agents/eve/browser-agent)
  tools for tasks that need the web.
- **Add skills.** Drop more `SKILL.md` procedures under `skills/`.
- **Swap the model.** Edit `agent/agent.ts`.
