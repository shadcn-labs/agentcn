The **Chat with PDF Agent** lets you ask questions about a PDF and get answers
grounded in the document, with a page citation for every claim. It indexes the
PDF into a vector store, retrieves only the relevant chunks per question, and can
also generate comprehension quizzes from real passages. Reach for it to turn
manuals, papers, and reports into something you can query.

<DocsBaseSwitcher base="eve" agent="chat-with-pdf" />

The **Eve** recipe runs the full RAG loop in one session: `index_pdf` chunks and
embeds the document into a libSQL vector store, then `search_docs` retrieves the
top matches for each question so the agent answers from real chunks — citing the
page each one came from.

## Install

```bash
npx shadcn@latest add https://agentcn.vercel.app/r/eve/chat-with-pdf
```

## Prerequisites

- An [Eve](https://vercel.com/docs/eve) project (`npx eve@latest init my-agent`).
  The recipe installs files under `agent/`, which Eve discovers automatically.
- The recipe's dependencies (`@libsql/client`, `unpdf`, `ai`, `@ai-sdk/openai`)
  install automatically with the shadcn CLI.
- An `OPENAI_API_KEY` for embeddings, and optionally `LIBSQL_URL` to point at a
  remote Turso database (defaults to a local file).
- For the **live preview below**, set `EVE_PREVIEW_URL` to a deployed Eve
  instance of this recipe. See the [Preview Backend](https://github.com/shadcn-labs/agentcn#preview-backend)
  notes in the README.

## Customization

- **Swap the vector store.** `lib/vector-store.ts` wraps libSQL — replace it with
  Pinecone, Qdrant, Chroma, or pgvector behind the same `upsertChunks`/`search`
  functions.
- **Tune chunking.** Adjust `chunkText`'s size and overlap for your documents.
- **Swap the embedding model.** Change the model in `lib/vector-store.ts`.
- **Add quizzes.** The instructions already support quiz generation from
  retrieved passages — extend them with your preferred format.
