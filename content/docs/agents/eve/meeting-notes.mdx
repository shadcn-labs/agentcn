The **Meeting Notes Agent** takes a raw meeting transcript and returns clean,
structured notes: a short summary, the decisions that were made, action items as
owner/task pairs, and any open questions. Reach for it to close out standups,
planning sessions, or customer calls without writing up notes by hand.

<DocsBaseSwitcher base="eve" agent="meeting-notes" />

The **Eve** recipe processes the transcript in a single session: it **reads**
the transcript, **extracts** the summary, decisions, and action items, then calls
the `save_notes` tool to emit a structured result — grounded only in what was
actually discussed.

## Install

```bash
npx shadcn@latest add https://agentcn.vercel.app/r/eve/meeting-notes
```

## Prerequisites

- An [Eve](https://vercel.com/docs/eve) project (`npx eve@latest init my-agent`).
  The recipe installs files under `agent/`, which Eve discovers automatically.
- The `eve` package (installed automatically as a dependency of the recipe).
- For the **live preview below**, set `EVE_PREVIEW_URL` to a deployed Eve
  instance of this recipe. See the [Preview Backend](https://github.com/shadcn-labs/agentcn#preview-backend)
  notes in the README.

## Customization

- **Reshape the notes.** Edit the schema in `save_notes` to add fields like
  attendees, risks, or next meeting date.
- **Persist the output.** Change `save_notes` to write to Notion, Linear, or a
  database instead of returning JSON.
- **Tune the extraction.** Edit `instructions.md` to change how owners are
  inferred or how decisions are phrased.
- **Swap the model.** Edit `agent/agent.ts` for longer transcripts or faster runs.
