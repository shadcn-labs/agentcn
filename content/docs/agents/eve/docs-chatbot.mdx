The **Docs Chatbot Agent** answers questions about a code library's API. It looks
up structured function documentation — signatures, parameters, and examples — and
answers using only what the docs say. Reach for it to drop a grounded "ask the
docs" assistant into a docs site, an IDE, or a support channel.

<DocsBaseSwitcher base="eve" agent="docs-chatbot" />

The **Eve** recipe ships a `lookup_docs` tool over a structured docs source
(`functions.json`). The agent retrieves matching functions and answers with the
signature and a usage example — and if nothing matches, suggests the closest
names instead of guessing.

## Install

```bash
npx shadcn@latest add https://agentcn.vercel.app/r/eve/docs-chatbot
```

## Prerequisites

- An [Eve](https://vercel.com/docs/eve) project (`npx eve@latest init my-agent`).
  The recipe installs files under `agent/`, which Eve discovers automatically.
- The `eve` package (installed automatically as a dependency of the recipe).
- For the **live preview below**, set `EVE_PREVIEW_URL` to a deployed Eve
  instance of this recipe. See the [Preview Backend](https://github.com/shadcn-labs/agentcn#preview-backend)
  notes in the README.

## Customization

- **Point at your docs.** Replace `data/functions.json` with your generated API
  docs (e.g. from TypeDoc), or change `lookup_docs` to fetch from a remote source.
- **Improve matching.** Swap the substring filter for embeddings-based search to
  handle paraphrased questions.
- **Expose it as MCP.** Wrap `lookup_docs` in an MCP server so other clients can
  query the same docs.
- **Swap the model.** Edit `agent/agent.ts`.
