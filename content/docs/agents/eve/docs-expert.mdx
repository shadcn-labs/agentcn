The **Docs Expert Agent** answers technical questions about libraries, APIs, and
tools by searching the live web — preferring official documentation and citing
every claim. Reach for it as a grounded "ask the docs" helper that returns
copy-pasteable answers instead of hallucinated APIs.

<DocsBaseSwitcher base="eve" agent="docs-expert" />

The **Eve** recipe ships a single `web_search` tool. The agent runs precise
queries, reads the results, and answers from official sources with a short code
example and citations.

## Install

```bash
npx shadcn@latest add https://agentcn.vercel.app/r/eve/docs-expert
```

## Prerequisites

- An [Eve](https://vercel.com/docs/eve) project (`npx eve@latest init my-agent`).
  The recipe installs files under `agent/`, which Eve discovers automatically.
- The `eve` package (installed automatically as a dependency of the recipe).
- An `EXA_API_KEY` for web search (the `web_search` tool calls the Exa API).
- For the **live preview below**, set `EVE_PREVIEW_URL` to a deployed Eve
  instance of this recipe. See the [Preview Backend](https://github.com/shadcn-labs/agentcn#preview-backend)
  notes in the README.

## Customization

- **Swap the search provider.** Edit `web_search.ts` for Brave, Tavily, or a
  docs-specific index.
- **Constrain to your docs.** Restrict queries to a domain to build a
  product-specific expert.
- **Add page fetching.** Add a `scrape_url` tool so the agent reads full pages.
- **Swap the model.** Edit `agent/agent.ts`.
