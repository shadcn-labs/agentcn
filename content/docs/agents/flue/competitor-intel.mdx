The **Competitor Intelligence Agent** takes a list of competitor URLs, scrapes
each page, and analyzes positioning, pricing, and key features. It returns a
single structured report you can drop into a dashboard, a sales brief, or a
weekly digest. Reach for it whenever you need a repeatable, grounded read on how
a set of competitors present themselves — without copy-pasting pages by hand.

<FrameworkDocTabs agent="competitor-intel" current="flue" />

The **Flue** recipe fans each page out to an `analyst` subagent. The
orchestrator **scrapes** every page to markdown with the `scrape_url` tool,
dispatches an analyst to extract positioning, pricing tiers, and notable
features — reporting only what is actually on the page — then **synthesizes** the
findings into a structured report through a bounded workflow.

## Install

npx shadcn@latest add https://agentcn.sh/r/flue/competitor-intel

## Customization

- **Swap the model.** Edit the `model` field in `agents/competitor-intel.ts`
  (and the workflow's orchestrator). Any model the Flue gateway supports works —
  for example `anthropic/claude-sonnet-4-6` for deeper analysis or
  `anthropic/claude-haiku-4-5` for speed.
- **Extend the tools.** Add another `defineTool` in `tools/` to enrich the
  report — e.g. a `screenshot` tool, a pricing-page crawler, or a CRM lookup.
  Wire new tools into the agent's `tools` array.
- **Add more URLs.** The workflow processes URLs sequentially, so you can pass as
  many competitors as you like — paste one URL per line in the preview above, or
  hand the workflow a longer `urls` array in code.
- **Reshape the report.** Adjust the `Report` valibot schema to capture
  additional fields such as funding, headcount, or sentiment.
