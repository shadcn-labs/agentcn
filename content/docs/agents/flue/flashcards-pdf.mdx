The **Flash Cards from PDF Agent** turns a document into ready-to-study flash
cards. It extracts the text, identifies the concepts worth memorizing, and writes
concise front/back pairs — and on request, generates an illustrative image for
each card. Reach for it to build study decks from textbooks, slides, or notes.

<DocsBaseSwitcher base="flue" agent="flashcards-pdf" />

The **Flue** recipe runs a workflow that returns a typed `Flashcards` object:
`parse_pdf` extracts the text, the agent drafts grounded front/back pairs, and
`generate_image` attaches an image URL per card when requested.

## Install

```bash
npx shadcn@latest add https://agentcn.vercel.app/r/flue/flashcards-pdf
```

## Prerequisites

- A [Flue](https://flueframework.com) project (`npx flue@latest init my-agent`).
- The `unpdf` dependency installs automatically with the shadcn CLI.
- An `OPENAI_API_KEY` if you want generated images (`generate_image` calls the
  OpenAI Images API).
- For the **live preview below**, set `ANTHROPIC_API_KEY`. The Flue preview runs
  in-process in this site's API route — no separate deployment required.

## Customization

- **Reshape the deck.** Edit the `Flashcards` valibot schema to add tags, hints,
  or difficulty.
- **Swap the image model.** Edit `generate-image.ts` or drop images entirely.
- **Export the deck.** Add a tool that writes to Anki, CSV, or your SRS.
- **Swap the model.** Change the `model` field on the agent.
