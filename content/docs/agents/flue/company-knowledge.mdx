The **Company Knowledge Agent** answers questions over your internal documents —
issues, wiki pages, notes. It indexes them into a vector store, searches the
corpus first, and redacts personal data (emails, phone numbers, SSNs, card
numbers, API keys) from responses. Reach for it to build an internal "ask the
company" assistant.

<DocsBaseSwitcher base="flue" agent="company-knowledge" />

The **Flue** recipe ships `search_knowledge` and `index_document` tools over a
libSQL vector store, a `redactPii` helper applied to retrieved passages, and an
`index-knowledge` workflow you can run on a schedule to keep the corpus fresh.

## Install

```bash
npx shadcn@latest add https://agentcn.vercel.app/r/flue/company-knowledge
```

## Prerequisites

- A [Flue](https://flueframework.com) project (`npx flue@latest init my-agent`).
- The recipe's dependencies (`@libsql/client`, `ai`, `@ai-sdk/openai`) install
  automatically with the shadcn CLI.
- An `OPENAI_API_KEY` for embeddings, and optionally `LIBSQL_URL` for a remote
  Turso/libSQL database (or swap in pgvector/Neon).
- For the **live preview below**, set `ANTHROPIC_API_KEY`. The Flue preview runs
  in-process in this site's API route — no separate deployment required.

## Customization

- **Connect sources.** Feed `index-knowledge` from Linear, Notion, or your wiki
  on a cron for fresh data.
- **Swap the vector store.** Replace `lib/vector-store.ts` with pgvector/Neon,
  Pinecone, or Qdrant.
- **Tune redaction.** Edit the patterns in `lib/pii.ts` for your data.
- **Swap the model.** Change the `model` field on the agent.
