The **Competitor Intelligence Agent** takes a list of competitor URLs, scrapes
each page, and analyzes positioning, pricing, and key features. It returns a
single structured report you can drop into a dashboard, a sales brief, or a
weekly digest. Reach for it whenever you need a repeatable, grounded read on how
a set of competitors present themselves — without copy-pasting pages by hand.

## How it works

The agent works through each URL sequentially. For every competitor it
**scrapes** the page to markdown with the `scrape_url` tool, then **analyzes**
the content to extract positioning, pricing tiers, and notable features —
reporting only what is actually on the page. Once every URL has been processed
it **synthesizes** the findings into a structured report with a top-level
summary. The Flue recipe additionally fans each page out to an `analyst`
subagent, while the Eve recipe runs the loop inside a single durable session.

## Customization

- **Swap the model.** In Eve, edit `agent/agent.ts`; in Flue, edit the `model`
  field in `agents/competitor-intel.ts` (and the workflow's orchestrator). Any
  model the framework's gateway supports works — for example
  `anthropic/claude-sonnet-4-6` for deeper analysis or
  `anthropic/claude-haiku-4-5` for speed.
- **Extend the tools.** Add another file under `agent/tools/` (Eve) or another
  `defineTool` in `tools/` (Flue) to enrich the report — e.g. a `screenshot`
  tool, a pricing-page crawler, or a CRM lookup. Wire new Flue tools into the
  agent's `tools` array.
- **Add more URLs.** Both recipes process URLs sequentially, so you can pass as
  many competitors as you like — paste one URL per line in the preview above, or
  hand the workflow a longer `urls` array in code.
- **Reshape the report.** Adjust the output schema (`synthesize_report` in Eve,
  the `Report` valibot schema in Flue) to capture additional fields such as
  funding, headcount, or sentiment.
